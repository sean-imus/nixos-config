# Dendritic NixOS Configuration Installation Guide

This guide walks you through installing this NixOS flake on a NEW machine.

## Prerequisites

- A target machine (physical or virtual)
- Access to clone your git repo on that machine

## Important: Machine-Specific Requirements

Two files are **machine-specific** and must be updated for each new machine:

1. **hardware-configuration.nix** - Generated by NixOS installer (contains disk UUIDs)
2. **hosts/default/default.nix** - Contains hostname (default: "nixos")

---

## Installation Steps

### Step 1: Initial NixOS Install (using ISO)

1. Download NixOS ISO from https://nixos.org/download.html
2. Boot from USB/CD in the target machine

3. Run the NixOS installer interactively:
   ```bash
   sudo nixos-install
   ```

4. Follow the prompts to set:
   - Root password
   - User account (sean + password)
   - Desktop environment (GNOME)

5. After install completes, **REBOOT** into the new system

---

### Step 2: Clone Your Config

After booting into the newly installed NixOS system:

```bash
# Clone your flake (replace URL)
git clone https://github.com/YOUR_USERNAME/nixos-config.git /etc/nixos
cd /etc/nixos
```

---

### Step 3: Get Machine-Specific Hardware Config

```bash
# Generate hardware config for THIS machine
sudo nixos-generate-config

# Copy it to your flake (overwrite machine-specific file)
sudo cp /etc/nixos/hardware-configuration.nix ./
```

---

### Step 4: Verify Hostname (if needed)

The current hostname in `hosts/default/default.nix` is `"nixos"`. If your machine has a different hostname:

```bash
# Check current hostname
hostname

# If different, edit hosts/default/default.nix and change:
# hostname = "your-actual-hostname";
```

---

### Step 5: Build and Test (Recommended)

```bash
# Test build (doesn't switch)
sudo nixos-rebuild build --flake .#nixos
```

If successful, you'll see the build output.

---

### Step 6: Apply Configuration (Final Step)

```bash
# Apply all changes
sudo nixos-rebuild switch --flake .#nixos
```

---

### Step 7: Reboot

```bash
sudo reboot
```

---

## Adding Another Machine Later

For a second/third machine:

1. **Create host directory**:
   ```bash
   mkdir -p hosts/<new-hostname>
   cp hosts/default/default.nix hosts/<new-hostname>/
   ```

2. **Edit hostname** in the new file:
   ```nix
   hostname = "new-hostname";
   ```

3. **Clone your flake** on that machine

4. **Generate hardware config**:
   ```bash
   sudo nixos-generate-config
   sudo cp /etc/nixos/hardware-configuration.nix ./
   ```

5. **Install**:
   ```bash
   sudo nixos-rebuild switch --flake .#<new-hostname>
   ```

## Common Commands

```bash
# Build (dry run - no switch)
sudo nixos-rebuild build --flake .

# Apply changes
sudo nixos-rebuild switch --flake .

# Update flake inputs (nixpkgs, etc.)
sudo nix flake update

# Roll back to previous generation
sudo nixos-rebuild switch --flake .#nixos --rollback

# List generations
sudo nixos-rebuild list-generations
```

## Understanding the Machine-Specific Files

### hardware-configuration.nix
Generated by `nixos-generate-config`. Contains:
- Boot kernel modules
- File system definitions (disk partitions)
- Swap device config

**Must regenerate** for each new machine because disk UUIDs are unique.

### hosts/default/default.nix
Contains:
- `hostname`: Must match your actual hostname
- User account config

**Update hostname** when copying to new machines.

---

## Troubleshooting

### "No such file or directory" for hardware config
→ Run `sudo nixos-generate-config` and copy the result

### Hostname doesn't match
→ Edit `hostname` in `hosts/default/default.nix`

### Build fails
→ Check error output - often missing hardware config or option typo

### "flake is dirty"
→ Commit changes or ignore (warning only, works anyway)
